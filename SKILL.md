# claude-session (cs) — постоянные Claude Code-сессии

Инструмент держит Claude Code запущенным в tmux-сессиях: по одной на проект,
они переживают закрытие терминала и перезагрузку сервера, а одна из них может
отвечать в Telegram. Пользователь может плохо разбираться в терминале и
ИИ-агентах — объясняй простыми словами, по одному шагу за раз, не вываливай
все команды сразу.

Как это устроено (для тебя, не пересказывай пользователю без нужды):
- `scripts/claude-session` (короткое имя `cs`) — CLI; список сессий в
  `~/.config/claude-sessions.conf` (формат `имя:папка[:channels]`).
- В каждой tmux-сессии `claude-restart-loop.sh` перезапускает Claude при
  падении и продолжает прошлый разговор (`claude -c`).
- systemd user unit `claude-sessions.service` поднимает все сессии при загрузке.
- Telegram-бот один: токен в `~/.claude/channels/telegram/.env`, и только одна
  сессия (с флагом `channels`) его обслуживает.

## Команды cs (справочник)

```
cs                          список сессий (имя, папка, telegram, запущена, подключён)
cs <имя>                    подключиться к сессии (сама запустится, если остановлена)
cs new <имя> <папка>        создать обычную сессию (без Telegram)
cs telegram <имя> [папка]   отдать сессии Telegram-бота (заберёт у прежней)
cs attach [имя]             подключиться; без имени — к единственной запущенной
cs rm <имя>                 убить сессию и убрать из конфига
```

Правило: `cs <имя>` / `cs attach` — интерактивные команды для человека
(захватывают терминал). Сам их не запускай — скажи пользователю набрать
команду в его терминале. Выход из сессии без её остановки: `Ctrl+B`, потом `D`.

## Сценарий A: первичная установка

Триггеры: «установи/настрой claude-session», «настрой cs», первый запуск.

0. Проверь, есть ли рядом с этим SKILL.md папка `scripts/`. Установщики вроде
   `npx skills add` копируют только SKILL.md — тогда сначала подтяни полный
   репозиторий и положи недостающее рядом (`$d` — папка этого SKILL.md):
   ```bash
   git clone --depth 1 https://github.com/mmmontov/claude-session "$d/.src" \
     && cp -r "$d/.src/scripts" "$d/.src/completions" "$d/" && rm -rf "$d/.src"
   ```
1. Запусти `scripts/install.sh` (лежит рядом с этим файлом). Он идемпотентен.
2. Разбери вывод. Блок «Требует внимания» — реши каждую проблему:
   - нет tmux → предложи установить (`sudo apt install tmux` или аналог для
     их системы) и после подтверждения выполни, затем перезапусти install.sh;
   - `~/.local/bin` не в PATH → предложи добавить строку в `~/.bashrc`.
3. Спроси, хочет ли пользователь управлять Claude через Telegram с телефона.
   Да → сценарий B. Нет → шаг 4.
4. Предложи создать первую сессию: спроси простыми словами, над каким
   проектом (папкой) человек работает и как назвать сессию (коротко,
   латиницей, например `main` или имя проекта). Затем `cs new <имя> <папка>`
   (или `cs telegram ...`, если Telegram уже настроен) и скажи, что
   подключиться можно командой `cs <имя>`.

## Сценарий B: подключение Telegram

Триггеры: «подключи телеграм», «хочу писать Claude с телефона», «настрой бота».

1. Включи плагин: в `~/.claude/settings.json` добавь
   `"enabledPlugins": {"telegram@claude-plugins-official": true}` (аккуратный
   merge — не затри остальные настройки; файла может не быть — создай).
2. Объясни создание бота по шагам, как для новичка:
   - открыть Telegram, найти `@BotFather` (синяя галочка);
   - отправить ему `/newbot`;
   - придумать отображаемое имя (любое, например «Мой Claude»);
   - придумать username бота — латиницей, обязательно кончается на `bot`
     (например `masha_claude_bot`);
   - BotFather пришлёт токен — длинную строку вида `1234567890:AA...`.
     Попросить прислать её тебе целиком.
3. Получив токен: попроси пользователя выполнить `/reload-plugins` (или
   перезапустить Claude Code), затем передай токен штатному скиллу плагина
   `telegram:configure`. Если тот недоступен — fallback вручную:
   - `~/.claude/channels/telegram/.env` (папку создать):
     `TELEGRAM_BOT_TOKEN=<токен>`, права `chmod 600`;
   - `~/.claude/channels/telegram/access.json`:
     `{"dmPolicy": "allowlist", "allowFrom": [], "groups": {}, "pending": {}}`.
4. Назначь сессию-обработчика: `cs telegram <имя> [папка]` (создастся, если
   не было). Объясни: бот живёт ровно в одной сессии; какая — видно в `cs`.
5. Пейринг: пользователь пишет своему боту `/start` в Telegram, затем
   одобри его через скилл `telegram:access`. После этого бот отвечает
   только этому человеку.
6. Предложи проверить: написать боту «привет» и убедиться, что он ответил.

## Сценарий C: повседневное управление

- «Переключись на сессию X» / «переключи бота на проект Y» →
  `cs telegram X` (если сессии нет — сначала выясни папку и создай).
  Предупреди: сессия, у которой бот был раньше, перезапустится без Telegram,
  а если ты сам работаешь в текущей telegram-сессии — после переключения
  сообщения в этом чате будет обрабатывать уже другая сессия.
- «Какие сессии есть / что запущено» → `cs` и перескажи по-человечески.
- «Создай сессию для <проект>» → `cs new`, подскажи `cs <имя>` для входа.
- «Удали сессию» → `cs rm <имя>`; если она держала Telegram — сразу предложи
  назначить бота другой сессии, иначе он замолчит.

## Если что-то не работает

- Бот молчит → `cs`: есть ли сессия с telegram и запущена ли она; затем
  `systemctl --user status claude-sessions`; проверь токен в
  `~/.claude/channels/telegram/.env` и что пользователь одобрен
  (`telegram:access`).
- Сессии пропали после перезагрузки → `systemctl --user enable claude-sessions`
  и `systemctl --user start claude-sessions`; на серверах без автологина
  может понадобиться `loginctl enable-linger <user>`.
- Claude в сессии постоянно перезапускается → подключиться (`cs <имя>`) и
  посмотреть, что он пишет; чаще всего слетела авторизация (`claude login`).
